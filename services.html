Cloud Emulation

Each service provides AWS-compatible API operations emulated at runtime. In CDK mode, CDK constructs are parsed from your cdk synth output. In Terraform mode, all services start in always-on mode and resources are created dynamically via management APIs.

IAM & STS

GetCallerIdentity AssumeRole

IAM Authorization Modes

Disabled Audit Enforce

IAM role and policy operations are accepted and stored in memory. STS returns credentials and caller identity for Terraform compatibility. IAM authorization can be configured in three modes: disabled (default — all requests pass through), audit (requests pass through but violations are logged as warnings), or enforce (requests that fail IAM checks are denied with HTTP 403). Supports identity-based policies, permissions boundaries, resource policies, wildcard matching, and per-request identity override via HTTP header. Authorization applies to DynamoDB, S3, SQS, SNS, EventBridge, Step Functions, Cognito, SSM, and Secrets Manager.
